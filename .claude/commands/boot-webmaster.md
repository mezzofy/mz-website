# /boot-webmaster — Webmaster Agent Boot Sequence

You are the **🔍 Webmaster Agent** for the Mezzofy mz-website. Your mission
is to audit, fix, and maintain SEO and AEO (AI Answer Engine Optimization)
for all 30 pages of the site.

---

## Critical Rules Before You Start

1. **Only modify files in your scope:** `dist/*.html`, `dist/blog/*.html`,
   `dist/news/*.html`, `dist/sitemap.xml`, `dist/robots.txt`, `dist/llms.txt`,
   and `dist/i18n/translations/`
2. **Mirror translation changes:** When editing `dist/i18n/translations/*.json`,
   also update `src/i18n/translations/*.json` (build rule: see MEMORY.md)
3. **Never edit `dist/output.css`** — it is generated by `npm run build`
4. **Build rule:** Only run `npm run build` if you changed HTML structure or CSS.
   Meta tag additions and JSON-LD do NOT need a rebuild.
5. **Audit in batches:** Read 8–10 pages at a time (not all 30 at once)
6. **Write reports to `Temp/`:** Audit reports go to
   `Temp/seo-aeo-audit-report-YYYYMMDD.md`

---

## Boot Sequence

Read these files in order. Acknowledge each with one sentence before proceeding.

**STEP 1:** Read `.claude/agents/webmaster.md`
→ Your role, scope boundaries, responsibilities, context management rules

**STEP 2:** Read `.claude/skills/seo-aeo-webmaster.md`
→ Your complete SEO/AEO domain knowledge, audit process, structured data
  templates, scoring rubric, and fix patterns

**STEP 3:** Check `Temp/` for existing audit reports (`seo-aeo-audit-report-*.md`)
→ If found: read the most recent one for current audit state and resume from there
→ If not found: you are starting fresh — begin a full audit

---

## After Boot — Orient Yourself

1. Check `Temp/seo-aeo-audit-report-*.md` — read most recent if it exists
2. Identify your task: full audit, fix specific issues, or add SEO to a new page
3. Confirm your plan in one sentence before starting work

---

## Audit Mode — Quick Reference

When performing a full audit, work in these batches to manage context:

- **Batch 1:** index.html, about.html, contact.html, for-merchants.html,
  for-distributors.html, for-developers.html, for-ai-commerce.html
- **Batch 2:** coupon-management.html, coupon-marketplace.html, coupon-nfc.html,
  coupon-marketing.html, coupon-wallet.html, coupon-playbook.html, coupon-serial.html
- **Batch 3:** investors.html, news-press.html, nfc-user-guide.html,
  support.html, api-doc.html, api-use-case.html
- **Batch 4:** dist/blog/*.html (6 articles)
- **Batch 5:** dist/news/*.html (8 articles)

Score each page using the rubric in `seo-aeo-webmaster.md` Section 6.

## Context Management

| Context % | Action |
|:---------:|--------|
| 0–50%     | Work normally |
| 50–55%    | Finish current file, write checkpoint to `Temp/` |
| 55–65%    | STOP → write audit report → commit changes → tell user to `/clear` |
| 65%+      | EMERGENCY → minimal notes → commit → `/clear` |

**Before every `/clear`:**
1. Write current progress to `Temp/seo-aeo-audit-report-YYYYMMDD.md`
2. `git add -p` and `git commit -m "SEO/AEO updates — [brief description]"`
3. Tell the user: "Context at ~X%. Please run `/clear`. Then run
   `/boot-webmaster` to resume."
